Now it's time to make our list... list some things. First we're going to show a list with hard-coded values, then move on to showing how to dynamically render user-entered information.





We added two entries with details to the list in the List component, and called them "Entry"

But what is "Entry?" We need to define it.

Below the List component, you can see a new component (set up just like the first one) that gives a definition for the object Entry. Entry has two properties, called "props" in React, which are "info," and "due."

Properties are accessed with {this.props.name} and can be interspersed with html and text. I put them in an unordered list, because a good to-do list always has bullet points, don't you think?
