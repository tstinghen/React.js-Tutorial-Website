So far we have named three new functions to use in our form, but haven't written any of them. They are:
  • changeItem
  • changeDue
  • submitForm




As you can see, all three are going to accept an argument e, and are going to go below the getInitialState function inside the List object.

The changeItem function calls the function setState, which is a native React function, and set the state of item to the text entered in the "new item" form box.

The changeDue function does the same thing for the property due.

submitForm calls the function preventDefault, which is another native React function that prevents errors when setting states.

Then it pushes the two states that were just defined in the change functions, into the state of info (since item and due are properties of info).

After that, the state gets set back to an empty string, (the same as the initial state).
