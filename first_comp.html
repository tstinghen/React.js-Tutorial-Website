We're going to write the first component. It's not going to do much, just print some text.

These first steps are very important in getting an understanding of how React operates and is structured. The first few actions that we will be working through could be easily accomplished in html, with a lot less effort. I hope that you will find that this set up is very much worth it, once we get to step 7 or 8, as we explore React's ability to interact with the user and create a virtual DOM.




The first thing to do is create a class of type React, which takes an object as it's parameter. We're going to call it "List" since it is going to be the basis of the entire to-do list... eventually.

Inside the object, we'll put a render function, and have it return some text and html. The render function creates something that we can print to the screen later.

The html or text being returned by the render function doesn't have to be in parenthesis, but if it is in parenthesis, we can add multiple lines of formatting marks, to make it look nice. This is where the concept of the Virual DOM really starts to come into play. The site is going to have formatted content, but the only thing outside the script tag is this single div. Pretty neat!

Next, we call a React function, React.renderComponent() and give it our object "List" and the location that we want it to render to (a basic html div in the body, called "toDoList") and it will print the contents of the render function to the div.
