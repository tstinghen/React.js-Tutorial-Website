We want to render our data dynamically, so we're going to need a component that takes some information and turns it into an Entry that is formatted for our to-do list.

This is a component that we are going to build now and use later. It's not going to have a back end yet, so it may not be obvious what it does or why we're making it until a few steps farther down the road, when we're actually getting input to create these new entries with newEntry.





It's time to create another component! This one is called newEntry, because it's going to get some information from... somewhere, and make it into an Entry.

newEntry could get information from an array of data, or from a file. We want it to come from the user, so we are not going to define where the information is coming from until later.

We map the information from info onto the object Entry, specifying what props map to what. We're going to make sure to call our info the same names as our Entry data, just to make things easier. That way, item maps to item, and due maps to due.







Since we're planning to have our information coming from this info object, which is creating entries for us, we need to make a change to the List component (that's the very first component we created.)

For right now, we are going to delete the to-do list info information from List. We'll add it back in later, once we've set up a system for getting the information we need.
